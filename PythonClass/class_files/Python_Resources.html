Editors & Development Environments

Sublime Text For class purposes I strongly recommend Sublime Text. Excellent support for Python, including syntax coloring and the ability to run a script and see the results within the editor.

PyCharm Powerful incredibly full-featured development environment. For serious projects, this is my favorite. It makes everything about Python even easier, including installing virtual environments and packages for you. Integrates with Git.

Atom A popular editor that I don't have personal experience with. If you want to use this, do a search for advice on how to best set it up for Python, as it doesn't do everything out of the box.

The Basics

Automate the Boring Stuff with Python This covers the basics of Python in more depth than we get to in class, and provides a number of interesting projects for typical tasks, like renaming files and interacting with PDFs.

learnpython.org online tutorial Using an embedded Python interpreter, this site takes you step by step through learning Python. It has nice features for showing you what you typed vs what was expected, asking for the answer, etc.

Web Scraping

Web Scraping with Python: Collecting Data from the Modern Web An excellent book with lots of code examples, covering the best libraries to use and techniques for getting data from websites, as well as how to set up an infrastructure for testing websites.

Being Pythonic

Fluent Python An excellent and very readable book delving into the topics that make Python special: iterators, generators, and the like.

Data Science

Python for Data Analysis: Data Wrangling with Pandas, NumPy, and IPython

jupyter notebooks These interactive notebooks are very popular for data science. They allow you to mix discussion and code, and to run and re-run specific bits of code, such as to regenerating a graph. They also support collaboration, and run on multiple platforms and in multiple languages.

Anaconda packages Some of the data science libraries are tricky to install. Anaconda solves this by providing pre-packaged libraries for all major platforms. They also include all the libraries typically used for data science.

A note about Python 2 vs 3

At Apple most teams and internal libraries use Python 2, so that's what we use for the class. Python books and tutorials have switched to Python 3, so you'll need to be aware of that. You can attempt to get an older version, or you can just handle the differences.

print is a function in Python 3

The most noticable difference that will impact you is that print is a function, so is called with parenthesis: print().

Many things return generators or views rather than lists

In Python 2, lists are frequently copied and returned from dictionaries and such. This is memory-intensive, so in Python 3 built-in objects have been changed to often return a generator or a "view" onto a list instead of the list itself. The impact on you is that an object will act like a list but then not support something you expect, like getting the length of the list.

If you need to treat such an object as a list, you can get a list version by calling list():

    mylist = list(list_like_object)

Just be aware that this will take up extra memory.
